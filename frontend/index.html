System Architecture

How the Collections Brain orchestrates Databricks capabilities to deliver AI-optimised collection strategies.

Data Flow

1. Ingest → Oracle (Lakeflow Connect CDC) + IceNet (SQL Server) + Zendesk (Streaming) + SFTP
2. Process → Medallion architecture (Bronze → Silver → Gold) in Unity Catalog
3. Feature Store → Customer 360 Feature Table with ~1,000 features
4. Train → Propensity-to-Pay + Best-Time-to-Contact models (MLflow experiment tracking)
5. Deploy → Register models in MLflow Registry, deploy to Model Serving endpoints
6. Serve → Real-time scoring via Model Serving with Feature Table auto-lookup
7. Orchestrate → Multi-Agent Supervisor calls models + MCP scorecard + RAG comms
8. Trace → Every agent decision logged in MLflow 3.0 for FCA audit compliance

ML & AI Components

🎯 Propensity-to-Pay Model
Tool: score_propensity_to_pay → MLflow Model Serving
ML model scoring likelihood of payment using ~1,000 features (employment, payment history, Open Banking data). Served in real-time via Model Serving with Feature Table auto-lookup.
Model Serving Docs MLflow on Databricks Feature Engineering
⏰ Best-Time-to-Contact Model
Tool: score_best_time_to_contact → MLflow Model Serving
Behavioural pattern analysis model predicting optimal day, time window, and channel for customer outreach. Uses employment patterns and historical contact success data.
Create Serving Endpoint MLflow Experiment Tracking
🤖 Multi-Agent Supervisor
Tool: recommend_strategy → Agent Framework
Orchestrates all tools: runs PTP model, BTC model, scorecard, vulnerability assessment, Open Banking check, then combines into an actionable strategy with compliance flags.
Agent Framework Create an Agent Foundation Model APIs
💬 Communication Generation (RAG)
Tool: generate_communication → Vector Search + LLM
Retrieves tone-appropriate communication templates from a Vector Search index, then personalises with customer data. Auto-selects tone (friendly/empathetic/formal/sensitive) based on vulnerability and arrears.
Vector Search RAG on Databricks

Data & Streaming Components

👤 Customer 360 Feature Table
Tool: lookup_customer → Feature Tables (Unity Catalog)
Unified customer profile combining demographics, account details, contact history, risk segments, and vulnerability flags. Materialised from Oracle loan system + IceNet + Zendesk via Lakeflow Connect.
Feature Tables in UC Unity Catalog
⚡ Real-Time Payment Signals
Tool: get_payment_history → Structured Streaming
Near-real-time ingestion of payment events, DD cancellations, and partial payments. Enables immediate signal detection (e.g., customer cancels direct debit → trigger proactive outreach).
Structured Streaming Delta Live Tables
🔌 Data Ingestion (Lakeflow Connect)
Oracle CDC + SQL Server + SFTP + Zendesk
Managed connectors for ingesting from Oracle (loan management), SQL Server/IceNet, SFTP (collections providers), and Zendesk (customer contacts). CDC-based for real-time, query-based for batch.
Lakeflow Connect Oracle Connector SQL Server Connector
🏦 Open Banking / ClearScore
Tool: get_open_banking_data → External API Integration
Verified affordability data from ClearScore/aggregators: income, expenses, disposable income, gambling detection. Used for FCA-compliant affordability assessments and payment plan recommendations.
Partner Connectors API Ingestion via Streaming

Governance & Compliance Components

🧾 Collection Scorecard (MCP Server)
Tool: get_scorecard_segment → MCP Server
Scorecard validation logic served as an MCP tool. Matches customers to collection segments (Early/Mid/Late/Pre-Litigation) with strategy, contact frequency limits, and escalation triggers.
MCP Server on Databricks Custom Agent Tools
🛡 FCA Vulnerability Engine
Tool: assess_vulnerability → Business Rules + Open Banking
Automated vulnerability screening per FCA Consumer Duty. Checks employment, arrears depth, engagement patterns, Open Banking signals (gambling), and existing flags. Determines handling approach.
Data Governance Access Control
📊 MLflow 3.0 Tracing (Audit Trail)
Every tool call → MLflow Trace
Full traceability of every agent decision — which models were called, what scores were returned, what strategy was recommended. Essential for FCA "Why did you prioritise this customer?" questions.

Production Roadmap

1
Data Ingestion Materialise Customer 360 from Oracle (via Lakeflow Connect CDC) + IceNet + Zendesk. Set up DLT pipelines for Bronze → Silver → Gold medallion layers.
2
Feature Engineering Create Feature Tables in Unity Catalog with ~1,000 features from historical collections data. Include Open Banking signals, ClearScore bands, and contact outcomes.
3
Model Training & Registry Train propensity-to-pay and best-time-to-contact models using MLflow experiment tracking. Register champion models in MLflow Model Registry with approval workflows.
4
Model Serving Deploy models to real-time Model Serving endpoints with Feature Table auto-lookup. Set up A/B testing between model versions.
5
Agent Orchestration Build Multi-Agent Supervisor using Agent Framework. Wire up MCP Server for scorecard, Vector Search for RAG comms templates. Enable MLflow 3.0 tracing for full audit trail.
6
Production App Deploy as a Databricks App with workspace SSO. Integrate with existing MSFT Agent / comms channels for WhatsApp, SMS, and email delivery. Target: >20% Cure/Roll uplift, 70hr/month automation savings.
